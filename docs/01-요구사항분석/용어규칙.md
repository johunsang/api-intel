# 용어 규칙 (Terminology Convention)

> **이 문서는 본 프로젝트의 모든 문서에서 반드시 준수해야 하는 용어 표기 규칙이다.**
> 문서 작성·수정 시 아래 규칙을 엄격히 따르며, 새로운 용어가 등장할 경우 이 문서에 먼저 등록한 후 사용한다.

## 1. 표기 원칙

| # | 원칙 | 올바른 예 | 잘못된 예 |
|---|------|----------|----------|
| 1 | **영문 약어**는 원문 그대로 표기한다. 한글 음차/번역하지 않는다 | `API`, `SDK`, `LLM` | ~~에이피아이~~, ~~엘엘엠~~ |
| 2 | **첫 등장 시** 약어의 풀네임을 괄호로 병기한다. 이후에는 약어만 사용한다 | "RAG(Retrieval-Augmented Generation) 기반…" → 이후 "RAG" | 매번 풀네임 반복 |
| 3 | **한글 용어**가 확정된 경우 한글을 주 표기로 하되, 첫 등장 시 영문을 괄호로 병기한다 | "환각(Hallucination)" | ~~할루시네이션~~ |
| 4 | **동일 개념에 복수 표현 금지.** 하나의 개념에는 하나의 용어만 사용한다 | "쿼리"로 통일 | ~~질의, 검색어, 질문~~ 혼용 |
| 5 | **서비스명은 정확한 형태만 사용한다.** 줄임말이나 변형 금지 | API Intelligence Engine | ~~API Intel~~, ~~AIE~~ |
| 6 | **복합어 띄어쓰기**는 용어 사전의 "표기" 열을 따른다 | "벡터 DB" | ~~벡터DB~~, ~~Vector DB~~ |
| 7 | **우선순위 표기**는 P0/P1/P2만 사용한다 | P0, P1, P2 | ~~높음/중간/낮음~~, ~~Must/Should/Nice~~ |

## 2. 용어 사전

### 2.1 서비스 고유 용어

| ID | 용어 (표기) | 영문 풀네임 | 정의 | 비고 |
|----|------------|------------|------|------|
| T-001 | API Intelligence Engine | - | 본 프로젝트의 서비스명. RAG 기반 API 의사결정 보조 플랫폼 | **정식 명칭. 줄임말 금지** |
| T-002 | 쿼리 | Query | 사용자가 시스템에 입력하는 자연어 요구사항 텍스트 | "질의", "검색어", "질문" 사용 금지 |
| T-003 | 인덱싱 | Indexing | 공식 문서를 크롤링하여 벡터 임베딩으로 변환·저장하는 과정 | |
| T-004 | 청크 | Chunk | 문서를 벡터 검색에 적합한 크기로 분할한 단위 텍스트 | 첫 등장 시 "청크(Chunk)" 병기 |
| T-005 | 리전 | Region | API 서비스가 운영되는 물리적 서버 위치/지역 | 첫 등장 시 "리전(Region)" 병기 |
| T-006 | 환각 | Hallucination | LLM이 사실이 아닌 정보를 자신 있게 생성하는 현상 | 첫 등장 시 "환각(Hallucination)" 병기 |

### 2.4 우선순위 표기

| 표기 | 의미 | 사용 맥락 |
|------|------|----------|
| P0 | Must Have | MVP에 반드시 포함. 이것 없이는 서비스가 성립하지 않음 |
| P1 | Should Have | MVP에 포함 권장. 핵심 경험을 위해 필요하지만 최소 기능은 아님 |
| P2 | Nice to Have | MVP에서 시간이 허용되면 포함. 없어도 핵심 가치 검증 가능 |

## 3. 용어 추가/변경 절차

1. 새로운 용어가 필요한 경우 → **이 문서의 해당 카테고리에 ID 부여 후 등록**
2. 표기 원칙(1절) 적용하여 표기 형태 확정
3. **프로젝트 내 모든 문서에 일관 반영** (기존 문서에 다른 표현이 있으면 수정)
4. 용어 삭제/변경 시 → 변경 이력에 기록 후 전체 문서 일괄 반영
